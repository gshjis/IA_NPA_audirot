# API Документация: Legal Semantic Search API

## Обзор
API предоставляет функционал для семантического поиска статей в юридических документах (Конституция, Трудовой кодекс и др.) с использованием предобученных моделей трансформеров.

## Эндпоинты

### `POST /search`
Выполняет семантический поиск статей на основе списка запросов.

#### Тело запроса (`SearchRequest`)
| Параметр | Тип | Описание | По умолчанию |
| :--- | :--- | :--- | :--- |
| `queries` | `Dict[str, str]` | Словарь запросов: `{id: текст_запроса}` | Обязательно |
| `k` | `int` | Количество возвращаемых статей (>= 1) | 5 |
| `similarity_weight` | `float` | Вес косинусного сходства (0.0 - 1.0) | 0.4 |
| `coverage_weight` | `float` | Вес покрытия тегов (0.0 - 1.0) | 0.6 |
| `penalty_factor` | `float` | Коэффициент штрафа за пропущенные теги (0.0 - 1.0) | 0.5 |

#### Пример запроса
```json
{
  "queries": {
    "q1": "право на труд",
    "q2": "защита персональных данных"
  },
  "k": 3,
  "similarity_weight": 0.5,
  "coverage_weight": 0.5,
  "penalty_factor": 0.3
}
```

#### Ответ (`List[SearchResponse]`)
Список объектов для каждого запроса:
- `request_id` (str): ID запроса из входного словаря.
- `results` (List[SearchResult]): Список найденных статей.

#### Структура `SearchResult`
- `article` (Dict[str, Any]): Оригинальные данные статьи.
- `score` (float): Итоговая оценка релевантности.
- `query_tags` (Dict[str, float]): Теги запроса и их веса.
- `article_tags` (Dict[str, float]): Теги статьи и их веса.
- `common_tags` (List[str]): Список общих тегов между запросом и статьей.

## Гиперпараметры ранжирования
Итоговая оценка (`score`) статьи рассчитывается по формуле:
`score = (similarity_weight * similarity + coverage_weight * coverage) - (penalty_factor * missing_tags_ratio)`

Где:
- `similarity`: Косинусное сходство векторов запроса и статьи.
- `coverage`: Доля веса запроса, покрытая тегами статьи.
- `missing_tags_ratio`: Доля тегов запроса, отсутствующих в статье.
