# LegalSemanticSearchEngine

Движок для семантического поиска и тегирования юридических текстов (статей Конституции) на основе векторных представлений.

## Математическая модель

### 1. Векторизация текстов

Тексты (статьи и описания тегов) преобразуются в векторы **v ∈ ℝᵈ** с помощью модели `SentenceTransformer`. Векторы нормализуются по L2‑норме:

```
v_norm = v / ||v||₂,   где ||v||₂ = √(∑ᵢ vᵢ²)
```

Нормализация гарантирует, что ||v_norm||₂ = 1, что упрощает вычисление косинусного сходства.

### 2. Косинусное сходство

Сходство между двумя нормализованными векторами **a** и **b** равно их скалярному произведению:

```
sim(a,b) = a·b = ∑ᵢ aᵢ bᵢ  ∈  [-1,1]
```

Поскольку векторы неотрицательны (модель выдает векторы с неотрицательными компонентами), сходство лежит в диапазоне [0,1].

### 3. Inverse Document Frequency (IDF)

Для учета информативности тегов вычисляется вес IDF на основе размеченного корпуса статей.

Пусть  
- **N** – общее число статей в корпусе,  
- **df(t)** – число статей, в которых встречается тег **t**.

Тогда:

```
idf(t) = log( N / (df(t) + 1) ) + 1
```

Добавление единицы в знаменатель предотвращает деление на ноль, а константа +1 смещает шкалу в положительную область.

### 4. Присвоение тегов статье

Для статьи **s** вычисляется её эмбеддинг **eₛ**. Для каждого тега **t** с эмбеддингом **eₜ** определяется сходство:

```
score_raw(s,t) = sim(eₛ, eₜ)
```

Статье присваиваются `tags_per_article` тегов с наибольшими сходствами.

### 5. Ранжирование статей по запросу

Запрос **q** (текст пользователя) преобразуется в набор тегов с весами:

```
w_q(t) = sim(e_q, eₜ) · idf(t),   t ∈ T_q
```

где **T_q** – топ‑`tags_per_article` тегов запроса.

Для каждой статьи **s** с известными тегами **Tₛ** и их весами **wₛ(t) = score_raw(s,t) · idf(t)** строится вектор по **общим тегам** **C = T_q ∩ Tₛ**:

```
v_q = [ w_q(t) ]_{t∈C},   vₛ = [ wₛ(t) ]_{t∈C}
```

Если **C** пусто, оценка статьи равна нулю. Иначе вычисляются:

1. **Косинусное сходство** общих векторов:

   ```
   sim_C = (v_q · vₛ) / (||v_q|| · ||vₛ||)
   ```

2. **Покрытие запроса** (coverage) – доля веса запроса, покрытая статьёй:

   ```
   cov = (∑_{t∈C} wₛ(t)) / (∑_{t∈T_q} w_q(t))
   ```

3. **Штраф за непокрытые теги**:

   ```
   r_miss = 1 − |C| / |T_q|,   penalty = 0.5 · r_miss
   ```

4. **Итоговая оценка** – взвешенная комбинация сходства и покрытия с вычитанием штрафа:

   ```
   score(s,q) = (0.4·sim_C + 0.6·cov) − penalty
   ```

Статьи сортируются по убыванию `score(s,q)`; возвращаются топ‑**k** уникальных статей.

### 6. Кэширование

Эмбеддинги тегов и статей сохраняются в файлы `.npy` в директории `data/cache`. При повторном запуске загружаются из кэша, что ускоряет инициализацию.

## Основные возможности

- **Автоматическое тегирование** – присвоение тегов статьям на основе семантического сходства.
- **Семантический поиск** – поиск статей по текстовому запросу с учётом IDF и взвешенного ранжирования.
- **Кэширование** – поддержка кэширования моделей и вычисленных эмбеддингов.

## Основные методы

### `__init__(tags_filepath, model_name, training_corpus, cache_dir, tags_per_article)`

Инициализирует движок: загружает теги, модель, вычисляет эмбеддинги тегов и IDF.

### `_compute_idf() -> Dict[str, float]`

Вычисляет IDF‑веса для всех тегов на основе размеченного корпуса.

### `assign_tags(articles_list, tags_per_article) -> List[Dict[str, Any]]`

Присваивает теги списку статей, используя косинусное сходство между эмбеддингами статей и тегов.

### `get_tag_recommendations(text) -> Dict[str, float]`

Возвращает словарь `{тег: оценка_релевантности}` для заданного текста.

### `find_articles_by_new_sentence(new_sentence, k=5) -> List[Dict[str, Any]]`

Находит топ‑**k** наиболее релевантных статей, используя описанную выше математическую модель ранжирования.

## Использование

```python
from src.search.engine import LegalSemanticSearchEngine

# Инициализация
engine = LegalSemanticSearchEngine(
    tags_filepath="data/raw/base.json",
    model_name='deepvk/USER2-base',
    cache_dir="data/cache"
)

# Поиск статей
results = engine.find_articles_by_new_sentence("Ваш запрос", k=5)
```
